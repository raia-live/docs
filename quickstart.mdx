## Get started

Set up your docs site in two steps — no configuration required to get started.

### Step 1: Create your account

Sign up using your existing Git provider credentials. No new password needed.

<CardGroup cols={2}>
  <Card title="GitHub" icon="github" href="https://dashboard.mintlify.com">
    Sign in with your GitHub account
  </Card>
  <Card title="GitLab" icon="gitlab" href="https://dashboard.mintlify.com">
    Sign in with your GitLab account
  </Card>
  <Card title="Bitbucket" icon="bitbucket" href="https://dashboard.mintlify.com">
    Sign in with your Bitbucket account
  </Card>
  <Card title="SSO / Email" icon="envelope" href="https://dashboard.mintlify.com">
    Sign in with SSO or email
  </Card>
</CardGroup>

<Tip>
  Your account is tied to your Git provider — use whichever you host your docs repository on.
</Tip>

### Step 2: Connect your repository

After signing in, link the repository that contains (or will contain) your documentation.

<Steps>
  <Step title="Open the dashboard">
    Go to [dashboard.mintlify.com](https://dashboard.mintlify.com) and click **New project**.
  </Step>
  <Step title="Select your repository">
    Grant Mintlify access to your Git provider and choose the repository you want to use. You can scope access to just that repository.
  </Step>
  <Step title="Configure your project">
    Set your project name and the directory where your docs live (defaults to the repo root).
  </Step>
  <Step title="Deploy">
    Click **Deploy**. Your docs site will be live at `yourproject.mintlify.app` within moments.
  </Step>
</Steps>

<Info>
  Every push to your default branch automatically redeploys your docs — no CI configuration needed.
</Info>

## Preview changes locally

Before pushing, you can preview every change live on your machine.

```bash
# Install the CLI
npm i -g mint

# Start the local preview server
mint dev
```

Open [http://localhost:3000](http://localhost:3000) — the preview hot-reloads as you edit files.
